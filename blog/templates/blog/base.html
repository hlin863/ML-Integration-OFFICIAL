Introduction

{% load static %}

Continuous Integration has come a long way since it's initial development in the 1990's. It's now a common practice in the software development industry and can be used to improve your code quality, increase productivity, and reduce the risk of bugs in your code. There are many areas where CI can be used, but in this tutorial we will focus on using it to help the user with integrating their commits on ML projects.

History of CI development

{% load static %}

The term continuous integration was first used in 1994 by Grady Booch. A first example when CI was applied was for the framework of an Extreme programming project "Chrysler Comprehensive Compensation System Project" in 1997. The project focuses on frequently integrating builds into the application.
